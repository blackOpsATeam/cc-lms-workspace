# Screen: Class, Batch & Subject Context вҖ” Class list

**Role(s):** Admin
**Density:** compact (admin convention вҖ” DESIGN.md В§13)
**Primary device:** desktop

## Purpose

Let an Admin browse every class in the institution, see status and operational health at a glance (mandatory subjects assigned, batch count), and start the create / edit / archive flows.

## Layout

Single-column desktop layout under the standard admin shell (top bar В§14.16 + left sidebar В§14.16). The screen body is one data table (В§14.12) preceded by a page header and a filter bar. No KPI band вҖ” the Class list is a working list, not a dashboard.

## Sections

### Page header

- Components: page title (`display.sm`), one-line description (`body.md В· text.muted`), primary CTA button (В§14.1) on the right.
- Tokens: `text.primary`, `text.muted`, `interactive.accent.*` for the CTA (amber, because creating a class is the highest-value action on this screen вҖ” В§2 / В§14.1 rule "one filled button per view section").
- CTA label: `+ New class`. On click вҶ’ opens the **Create class** dialog (В§14.13, sm width 400px).

### Filter bar

- Components: inline search (В§14.17, `md` size), status select (В§14.3), result count caption.
- Search debounced; partial match on `Class.name` and `Class.code`. Clear (Г—) shows when value present.
- Status select options: **All**, **Active**, **Inactive**, **Archived**. Default: **Active** (admins almost always want the live set first).
- Result count uses `body.sm В· text.muted В· tabular`. Updates as filters change.
- Tokens: `surface.raised` for the bar background, `border.subtle` divider below.

## States

### Empty

- Trigger: institution has 0 classes (first run after Admin onboarding).
- Layout per В§14.18: centered, 48px padding.
- Icon: `GraduationCap` (Lucide) В· 32px В· `text.muted`.
- Heading: "No classes yet" (`heading.sm`).
- Body: "Create your first class to start defining batches and assigning subjects." (`body.sm В· text.muted`).
- Action: **filled navy primary button** `+ New class` (md size, В§14.1). The amber variant is reserved for the page-header CTA, which stays visible вҖ” only one filled amber button may exist on a screen (В§2, В§14.1). The empty-state action shares the same target.

A separate **filtered-empty** state: "No classes match these filters." with a `Clear filters` ghost button (В§14.1).

### Loading

- Skeleton (В§14.19) matching the populated layout:
  - 1 page-header row + filter bar are real (controls remain interactive).
  - Table body shows 8 skeleton rows. Each row: 7 skeleton blocks matching column widths, `surface.sunken`, ~1.5s pulse, 36px row height.

### Populated

- Default state (see ASCII).
- Result count and pagination caption reflect filtered total.

### Error

- Trigger: list fetch fails.
- Render an inline alert *above* the table (not a toast вҖ” В§14.14 rule "use inline alert for anything the user must act on"):
  - Surface: `surface.raised` + 3px left border in `status.cancelled.fg` (lifecycle family, В§5 вҖ” the only semantic danger expression available without introducing a new token).
  - Icon: `AlertCircle` 20px В· `status.cancelled.fg`.
  - Title: "Couldn't load classes." (`body.md В· 500 В· text.primary`).
  - Body: server error message (`body.sm В· text.secondary`).
  - Action: `Retry` secondary button (В§14.1 sm).
- Table area below collapses to a single empty row stating the same.

## Interactions

- **Tap on row (anywhere except вӢҜ)** вҶ’ navigate to **Class detail** (subjects + batches tabs).
- **Tap on вӢҜ button** вҶ’ opens a **context-action menu popover** (not a `<select>` element). It borrows visual dimensions from В§14.3 вҖ” `surface.overlay`, `radius.md`, `shadow.md`, layer `z.dropdown`, option rows 36px high with 12px horizontal padding вҖ” but is implemented as a Menu / Popover anchored to the вӢҜ button, with keyboard navigation per the standard menu pattern. Options depend on current `status`:
  - `ACTIVE`: `Edit`, `Deactivate`, `Archive`
  - `INACTIVE`: `Edit`, `Activate`, `Archive`
  - `ARCHIVED`: `View` only (no edit, no un-archive вҖ” `ARCHIVED вҶ’ *` is rejected per В§11 SRS Validation Rules)
- **Tap `+ New class`** вҶ’ opens Create class dialog (separate spec).
- **Tap `Archive`** вҶ’ opens a confirm modal (В§14.13 sm). If the class has non-archived batches, the modal shows the error inline before confirm: "This class has 3 non-archived batches. Archive its batches first." (FR-CBS-005) вҖ” Confirm is disabled, only Cancel is enabled. Otherwise: title "Archive this class?", body "Archived classes can't have new batches. Existing batches keep operating.", actions: Cancel (secondary) В· `Archive` (destructive variant, В§14.1).
- **Tap `Activate` / `Deactivate`** вҶ’ optimistic toggle with success toast (В§14.14, 5s auto-dismiss). On failure, revert and show inline alert.
- **Keyboard:** rows are focusable; Enter opens the row. The вӢҜ button is reachable via Tab and Space/Enter opens the menu. Focus ring per В§14.0 (2px `border.focus`, 2px offset).

## Data needs

- **Endpoint:** `GET /classes?status=<ACTIVE|INACTIVE|ARCHIVED|ALL>&q=<search>&page=<n>&page_size=25` вҖ” *to be specified in `docs/api-contracts/class-context.md`*. Returns array of `{ id, name, code, status, subject_count, batch_count, updated_at }` plus `{ total, page, page_size }` envelope. Subject count = mandatory `ClassSubject` rows; batch count = non-archived `Batch` rows. Tenant-scoped via JWT (per SRS В§8 multi-tenancy block).
- **Status change endpoint:** `POST /classes/{class_id}/status` (already defined, SRS В§9). Used by Activate / Deactivate / Archive row actions.
- **Refresh:** on mount; on filter / search / pagination change; after any row-level write (optimistic, with revalidation).
- **Cache:** not in resolution cache scope (that's per-batch only); a plain SWR/RTK-Query list cache is sufficient.

## Bangladesh-context

- **Names:** class names may be Bangla ("аҰЁаҰ¬аҰ® аҰ¶а§ҚаҰ°а§ҮаҰЈаҰҝ"). Wrap Bangla substrings in `font.bn` per В§12. Allow row height to flex if a long Bangla name wraps.
- **Codes:** ASCII only by validation (В§11 SRS) вҖ” always render in `font.mono`.
- **Dates:** `11 May 2026` (В§15). Western Arabic numerals only, even in Bangla content (В§12).
- **No currency on this screen.**

## Accessibility

- **Touch targets:** rows are 36px tall (compact, В§7 / В§14.12) вҖ” desktop's permitted minimum is 32 Г— 32px (mouse-precision context, В§7), so the row and the вӢҜ button (32 Г— 32px) both clear that bar. This screen is desktop-primary; if a future mobile variant is added, it must switch to the comfortable 48px row height to meet the 44 Г— 44px mobile minimum.
- **Color-alone rule (В§18):** status is communicated by both the colored pill *and* the literal status label inside it (ACTIVE / INACTIVE / ARCHIVED). The "no subjects assigned" warning uses the `AlertTriangle` icon shape, the colored cell content, **and** the visible inline label "No subjects" вҖ” three independent signals. The icon also carries `aria-label="No mandatory subjects вҖ” batches under this class cannot be activated (FR-CBS-012)"` for screen readers.
- **Focus order:** Page CTA вҶ’ search вҶ’ status filter вҶ’ table header (sortable) вҶ’ row 1 вҶ’ row 1 вӢҜ вҶ’ row 2 вҶ’ вҖҰ вҶ’ pagination.
- **Sticky header** does not steal focus during keyboard navigation.
- **`aria-label`** on вӢҜ button: "Actions for class {name}".
- **Screen reader:** table uses proper `<table>` semantics with `<th scope="col">` and `<caption>`. Bangla cells carry `lang="bn"` (В§18).

## Out of scope

- Adding subjects to a class (that's the Class detail spec вҖ” Subjects tab).
- Creating / editing / managing batches (Batch list and Batch detail specs).
- Bulk operations (multi-select archive, etc.) вҖ” v1 is single-row only.
- Import / export of classes вҖ” not in v1 scope.
- Teacher and student counts at the class level вҖ” not in the SRS; would require a derived count outside `Class`. Skip in v1.
